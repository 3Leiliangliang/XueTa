## 3. 已完成内容

### 3.1 前端（`Web/`）

已完成并接入后端的页面与能力：

- 路由已覆盖：`/translate`、`/qa`、`/note`、`/kb`、`/practice`、`/progress`、`/profile`、`/desktop`、`/planning`、`/auth/*`、`/about`
- 登录/注册/重置密码流程可用，Token 持久化可用
- 个人中心已接 `users/me`，支持资料修改与登出
- 规划页已接目标/任务 CRUD，AI 生成后可刷新并展示持久化任务
- 笔记页已接笔记本、笔记、待办、总结
- 问答页已接会话、消息历史、SSE 流式接口
- 翻译页已接文本翻译、润色、文件链路与保存到笔记
- 知识库页已接知识库/文档/切块/检索
- 练习页已接练习生成、作答提交、错题与历史结果
- 进度页已接概览、掌握度、学习记录、复习计划
- 桌面页已接布局云端保存，并保留本地兜底

### 3.2 后端（`backend/`）

已完成的核心模块：

- `auth`：注册、密码登录、验证码登录、刷新 Token、登出、忘记密码、重置密码
- `users`：当前用户查询与资料更新
- `planner`：目标/任务 CRUD + AI 计划生成
- `notes`：笔记本、笔记、待办、总结
- `chat`：会话、消息、反馈、流式接口
- `translate`：文本翻译、润色，支持文件/URL 内容提取后翻译
- `kb`：知识库/文档/切块 CRUD 与检索
- `practice`：题集生成、作答、评分、错题沉淀
- `progress`：学习记录、掌握度、复习计划、概览统计
- `desktop`：布局按名称读写与列表管理
- `files`：上传、列表、详情、下载、删除
- `tasks`：Redis 队列 + worker + 注册机制（已可跑 demo 任务）

### 3.3 AI 与检索能力

已实现：

- OpenAI 调用封装（聊天、翻译、润色、笔记总结）
- OpenAI 不可用时 fallback 逻辑（规则/哈希向量）
- 聊天 SSE 流式输出（支持真实流式与fallback）
- 知识库 embedding 生成与存储
- 检索支持向量 + 关键词混合打分
- PostgreSQL 下支持 `pgvector` 距离检索，非 PG 环境支持 Python 余弦回退

### 3.4 文本抽取与多源输入

`content_extraction` 已支持：

- `txt/md/csv/json/js/ts/html`
- `docx`
- `pdf`（含无 `pypdf` 的 fallback 解析）
- `image`（Tesseract/OpenAI OCR 兜底）
- URL 正文抓取与抽取

### 3.5 工程化与部署

已完成：

- `docker-compose.yml` 全栈编排：`postgres + redis + backend + worker + web`
- 后端 Alembic baseline 已存在：`20260407_0001_baseline_schema.py`
- 后端自动化测试通过（27 项）
- 前端当前测试与构建通过

## 4. 未完成与待完善项

### 4.1 产品体验层

- 全局悬浮 `AIAssistant` 仍是本地模拟问答，未接后端真实能力
- `qa.vue` 中朗读/收藏仍是 TODO
- 前端尚未形成完整的 token 自动刷新与重试机制（401 时主要清理会话）
- 部分页面仍有静态展示信息（如部分桌面小组件内容）

### 4.2 AI 质量层

- 练习生成与评分仍偏规则型，AI 质量与稳定性仍需升级
- 聊天与知识库联动虽已具备基础，但引用质量、重排、可解释性仍可增强
- 提示词、模型选择、失败重试、限流策略仍缺统一策略层

### 4.3 任务队列与异步化

- `tasks` 模块已搭好框架，但业务任务仍以 demo 为主
- OCR、文档解析、embedding、批处理尚未全面迁移到异步队列

### 4.4 质量保障与发布体系

- 前端测试覆盖仍少（目前仅工具函数测试）
- 缺少前后端联调自动化与 E2E
- CI 流程（lint/test/build/镜像）尚未形成完整闭环

### 4.5 文档一致性与编码质量

- 部分文档描述已落后于当前代码（如旧描述仍把某些能力写成“未实现”）
- 代码与文档中存在中文乱码（编码不一致）问题，影响可维护性与团队协作

## 5. 后续开发方案（建议分阶段推进）

### 阶段 A：稳定性与一致性（1-2 周）

目标：先提升“可维护、可协作、可持续开发”能力。

建议任务：

1. 修复项目中文编码（统一 UTF-8，无 BOM）
2. 统一整理 README/ARCHITECTURE/模块文档，确保与代码一致
3. 完成前端 token 自动刷新链路（`auth/refresh`）
4. 将全局 `AIAssistant` 改为真实后端会话模式，或临时下线避免误导
5. 收敛 UI TODO（QA 朗读、收藏占位逻辑）

### 阶段 B：异步任务化与 RAG 强化（2-4 周）

目标：把重任务从同步 API 拆出去，提升稳定性与吞吐。

建议任务：

1. 将文档解析、OCR、embedding 迁移到 `tasks` 队列
2. 为任务提供状态查询、重试、超时、告警
3. 增加检索重排（rerank）与引用展示增强
4. 优化检索召回参数与打分策略（关键词/向量权重）
5. 补充知识库导入链路监控（成功率、耗时、失败类型）

### 阶段 C：AI 质量升级（2-4 周）

目标：从“能用”提升到“好用、稳定、可信”。

建议任务：

1. 练习生成升级为模型驱动（含题型约束与难度控制）
2. 练习评分改为“规则 + 模型”混合评估
3. 聊天回答升级为结构化输出（要点、步骤、引用）
4. 增加模型失败分级兜底策略（降级而非直接失败）
5. 接入 Langfuse 做 prompt/trace/结果质量观测

### 阶段 D：测试与发布体系（并行推进）

目标：保证每次迭代可回归、可发布。

建议任务：

1. 新增前端页面级测试（至少覆盖 kb/practice/progress/profile）
2. 新增后端 `auth/users/tasks` API 测试
3. 增加端到端关键路径测试（登录 -> 学习 -> 练习 -> 进度）
4. 落地 CI：`backend pytest + web vitest + web build`
5. 增加发布前检查清单（配置、迁移、健康检查、回滚）

## 6. 建议的近期优先级（P0/P1/P2）

### P0（立即）

1. 统一编码并修复乱码
2. 前端接入 token 自动刷新
3. 文档与代码状态对齐

### P1（本迭代）

1. `tasks` 承接文档解析/OCR/embedding
2. QA TODO（朗读/收藏）补齐
3. 增强 RAG 结果展示（引用来源+高亮）

### P2（下一迭代）

1. AI 练习生成与评分升级
2. Langfuse 全链路观测
3. E2E + CI 完整上线

## 7. 当前结论

XueTa 当前阶段已经不是“演示原型”，而是：

- 前后端主业务可联调运行
- 关键学习闭环已具备（目标/任务 -> 学习行为 -> 练习 -> 进度反馈）
- 具备继续向产品化推进的基础架构

后续最值得投入的主线是：

1. 先把稳定性与一致性（编码、文档、鉴权链路）做扎实
2. 再把重能力迁移到异步队列并强化 RAG
3. 最后集中做 AI 质量与测试发布体系
